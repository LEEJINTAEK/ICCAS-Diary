## 게임 개발

<br />

많은 ZOOM 회의 끝에 우리의 개발 일정, 개발 환경, 게임 시나리오, 역할 분담이 정해졌다. <br />

<img src="https://github.com/LEEJINTAEK/ICCAS-Diary/assets/109197023/93a12cdd-2f76-4b90-bb38-c391cb1801fd" alt="Image" style="width: 700px; height: 320px;">

<br />

개발 환경

```plaintext
1. Git 협업
2. Unity 개발
3. Firebase 데이터베이스
```

<br />

우리 팀은 다들 Git은 조금이나마 사용할 줄 알았고, <br />
**개발 3, 디자인 1** 로 균형이 잘 맞춰져 있었다!!

<br />

Git 협업 시에 우린 **Collaboration** 을 추가하여 <br />
**각자의 name branch -> develop -> main 의 방식의 전략**을 이용하였다

각자 Git 협업은 낯설기 때문에 최대한 충돌을 피하고자 소통을 많이 했고, <br />
최대한 개발 부분도 안 겹치게 진행했다.

### 프로젝트 발표

교육 기간 중, 1차 프로젝트 발표가 있었다. <br />

<img src="https://github.com/LEEJINTAEK/ICCAS-Diary/assets/109197023/ca787b07-0f93-42bd-808d-23b81e9f8f36" alt="Image" style="width: 500px; height: 290px;">

<br />

내 인생 첫 영어 발표였지만, 무사히 끝냈다. <br />
하지만!! 교수님들의 질문과 피드백을 어마어마하게 받았다.<br />

기억나는 피드백은..

"게임을 진행하며 환자를 위한 요소를 넣는 것이 어떤지, ai를 어떻게 하면 잘 사용할지" 였다. <br />

```plainText
피드백 후 팀 회의

1. 지능형 게임이다 보니까, ai를 넣으려고 했다. <br />
2. 다음 점수를 예측하는 ai였다.
3. 사용자의 데이터가 없었기에 머신 학습에 무리가 있었다.
4. 그렇다면 Chat gpt로 환자에게 격려하는 메세지를?!
5. 그러나 그것은 ai 필요 없이 구현 가능..한데...
```

<br />

고민을 정말 많이 했는데, 지도교수님이

- "지능형이라고 해서 꼭 ai를 살릴필요 없다."
- "지금 프로젝트에는 ai 안 넣는 것이 더 자연스럽다"

라고 의견을 남겨주셨다!! 따라서, 깔끔하게 ai를 포기하고 환자를 위한 설문조사와, 요소들을 추가 개발을 했다.

<br />
<br />

시간이 지나, 몰입 교육 마지막 날! <br />
영어 스피킹과 영어 발표 레슨을 받고 끝이 났다. <br />

개발한 게임을 main에 merge했다. 그리고 프로젝트 ppt, 데모 영상, 포스터 제작까지 완료 했다. (독일 갈 준비 끝!! 👊)

### 게임 시연

<br />

독일 뮌헨의 '마리엔 광장' 에서 제작한 게임을 시연해보는 날이었다. <br />

<img src="https://github.com/LEEJINTAEK/ICCAS-Diary/assets/109197023/6d8499b7-f0a2-4f12-91db-381b1a7baeff" alt="Image" style="width: 500px; height: 280px;">

<br />

제작한 게임은 pc 버전인데, 이러쿵저러쿵 하는 사연 때문에 <br />
게임데모영상으로 현지 사람에게 시연했다. <br />
(사람들이 시연 요청을 거절하는 바람에 겨우 피드백을 받을 수 있었다.)

공통적인 의견
`"게임 영상으로 봤을 땐, 강박증 환자들이 게임을 진행하면서 불안 증상이 악화될 것 같다"` 였다. <br />

하지만 이것은 국내에서 피드백을 받았던 내용이였고, <br />
"환자들의 불안감을 덜 수 있게 '난이도 조절기능', '격려 메세지 애니메이션' 을 추가하였습니다."의 설명과 함께 <br /> 관련 영상을 보여주었다.<br />

그러고 나서, 긍정적인 반응을 얻을 수 있었다.!! <br />

나는 이 경험을 통해,

1. 사용자 피드백의 중요성
2. 프로젝트 개선의 필요성
3. 즐겁다.

라는 느낌을 받을 수 있었다.

## 마무리

<br />

<img src="https://github.com/LEEJINTAEK/ICCAS-Diary/assets/109197023/d864abc5-72f7-4b4a-9dea-2f6668ed0cab" alt="Image" style="width: 500px; height: 280px;">

<br />

프로젝트 시작부터 끝까지 느꼈던 점을 간단히 요약했다.

1. 협업

- 다양한 의견 충돌이 있었지만, 수렴하고 수용하면서 성공적인 개발을 할 수 있었다.
- 협업 tools 를 잘 사용하자

2. 나의 첫 게임 개발

- 나에게 도전적이였던 C#과 Unity 금방 배운다!(나는 아마 야매..?) 👉[**미니프로젝트 저장소**](https://github.com/LEEJINTAEK/Unity_Games)
- 게임 개발자는 천재인가?

3. 나의 첫 해외 발표

- 영어 발표는 문제 없지만, 답변할 때 문제가 있었다. (스피킹 공부가 필요하다)

4. 사용자의 피드백

- 사용자들의 피드백이 없다면, 그 프로젝트는 발전할 수 없는 것 같다.

5. 개발자의 길

- 개발 과정과, 사용자에게 시연하는 과정은 너무 즐겁고 재밌었다.
- 개발자로서 성장해야겠다.

6. 독일

- 나중에 해외 개발자로서 경험을 해보고 싶다.
